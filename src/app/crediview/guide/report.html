디자인시스템 코드 변환기
css class 명명 규칙
기본 규칙

· BEM 명명규칙을 변형하여 사용

· 단어의 연결은 하이픈을 사용하며, 언더스코어는 사용하지 않는다.

· "block-element" (블록-요소)

· "block-element-element" (블록-요소-하위요소 최대 3단)

· "key--value" (변형요소--변형값)

· "block-element key--value" (블록-요소 변형요소--변형값 조합)

· ex: "SOLButton-root s--xl"
Component

· prefix : SOL[컴포넌트명]-

· 컴포넌트 최외곽 : SOL[컴포넌트명]-root

· ex: SOLInput-root > SOLInput-label
css 속성 선언 규칙
기본 규칙

· 레이아웃-box-배경-폰트-동작의 순서로 선언한다.

· z-index 속성은 최상단에 작성한다. (가상선택자의 경우에는 content 속성 다음에 작성)

· 미디어 타입은 각 파일 하단에 모아서 작성한다.

· 선택자의 중첩 선언은 최대 3단까지만 허용한다.
